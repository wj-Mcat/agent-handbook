# CLAUDE.md

本文件为 Claude Code / AI 编程助手在本仓库中工作的项目指南。请在修改代码、文档或站点配置前优先阅读并遵守这些约定。

## 项目概览

这是一个面向大语言模型、Agent、提示词工程与论文阅读的知识学习笔记本，基于 Docusaurus 构建为静态文档站点。

核心定位：

- 以中文为主记录 LLM / Agent 学习笔记、技术框架、论文解读和实践经验。
- 使用 Docusaurus 的多文档实例组织内容：默认 docs 文档区 + 独立 llms 文档区 + blog。
- 支持 Markdown / MDX、数学公式、React 组件扩展、Giscus 评论和 GitHub Pages 部署。

## 技术栈

- Node.js：要求 >= 20.18.1
- Docusaurus：3.4.0
- React：18
- TypeScript：~5.2.2
- Markdown / MDX：文档主体格式
- remark-math + rehype-katex：数学公式支持
- @giscus/react：评论组件
- antd：可用于 React UI 组件
- clsx：条件 className 拼接
- GitHub Actions + GitHub Pages：自动构建部署

## 常用命令

```bash
npm install
npm run start
npm run build
npm run serve
npm run typecheck
npm run clear
```

说明：

- `npm run start`：本地开发，端口为 3001。
- `npm run build`：生成静态站点到 `build/`。
- `npm run serve`：本地预览构建产物，端口为 3001。
- `npm run typecheck`：运行 TypeScript 类型检查。
- 当前仓库存在 `package-lock.json`，优先使用 npm；GitHub Actions 当前使用 yarn，若要统一包管理器，需要单独确认后再调整。

## Docusaurus 配置要点

主配置位于 `docusaurus.config.ts`：

- 站点标题：`《Agent Handbook》`
- tagline：`Agent Is All You Need`
- 默认语言：中文 `zh`
- 生产地址：`https://wj-mcat.github.io/`
- baseUrl：`/agent-handbook`
- 默认 docs 插件使用 `docs/`，侧边栏为 `sidebars.ts`
- 额外 docs 插件 id 为 `llms`，路径为 `llms/`，路由为 `/llms`，侧边栏为 `sidebarsLlms.ts`
- 额外 docs 插件 id 为 `paper-reading`，路径为 `paper-reading/`，路由为 `/paper-reading`，侧边栏为 `sidebarsPaperReading.ts`
- 额外 docs 插件 id 为 `weekly-paper`，路径为 `weekly-paper/`，路由为 `/weekly-paper`，侧边栏为 `sidebarsWeeklyPaper.ts`
- Markdown/MDX 均启用 `remark-math` 与 `rehype-katex`
- `onBrokenLinks` 当前为 `ignore`，`onBrokenMarkdownLinks` 当前为 `warn`

修改配置时要注意：

- 不要随意改变 `baseUrl`、`url`、`deploymentBranch`，这些会影响 GitHub Pages 部署。
- 新增文档区时，应同步更新 `docusaurus.config.ts`、对应 sidebar 文件和导航栏。
- 新增数学公式内容时，优先使用已有 KaTeX 支持，不要重复引入新的公式渲染库。

## 内容组织约定

### 默认 docs 文档区

`docs/` 下当前主要包含：

- `00-agent-intro/`：Agent 介绍、记忆、工具使用、规划、工作流
- `01-llm-intro/`：大语言模型、Transformer、Function Call、训练等
- `02-prompt-engineering/`：ICL、CoT、任务分解、集成方法、SFT 等
- `03-agent-application/`：数据生成、RAG、OpenAI o1、Mobile Agent 等

默认 docs 的侧边栏在 `sidebars.ts` 中配置，主要通过 `autogenerated` 从目录结构自动生成。

### paper-reading 文档区

`paper-reading/` 是单独的文档实例，路由为 `/paper-reading`，侧边栏在 `sidebarsPaperReading.ts` 中配置。

当前结构按年份/月份组织论文领读笔记，例如 `2024/07/self-play-with-execution-feedback.md`。

### weekly-paper 文档区

`weekly-paper/` 是单独的文档实例，路由为 `/weekly-paper`，侧边栏在 `sidebarsWeeklyPaper.ts` 中配置。

`index.md` 为周刊总索引页，各周文档如 `week-1.md` 按周归档。

### llms 文档区

`llms/` 是单独的文档实例，路由为 `/llms`，侧边栏在 `sidebarsLlms.ts` 中配置。

当前结构：

- `00-intro.md`：LLMs 发展历程入口页，slug 为 `intro`
- `01-basics/`：基础知识，如 Tokenization、Transformer、Attention
- `02-training/`：训练相关内容，如预训练和数据流水线

### blog 博客区

`blog/` 按年份和月份组织文章，例如：

```text
blog/2025/03/paper-reading-gen-tool.md
```

博客元信息通常包含：

```markdown
---
slug: article-slug
title: 文章标题
authors: wj-Mcat
tags: [Paper Reading, tools, Function Call, Agent]
toc_min_heading_level: 2
toc_max_heading_level: 5
---
```

## 文档写作规范

本仓库是中文知识笔记本，新增或修改内容时请遵守：

- 默认使用中文撰写正文，保留必要英文术语，如 Agent、LLM、Function Call、RLHF、RAG。
- 标题层级从 `#` 开始，正文中使用 `##`、`###` 组织结构。
- 文件名建议使用小写英文、数字和连字符，便于 URL 稳定，例如 `data-curriculum.md`。
- 目录排序可使用数字前缀，例如 `00-intro.md`、`01-dataset.md`。
- 文档页可使用 front matter 设置 `sidebar_position`、`slug`、`comments`。
- 图片优先放在当前主题目录下的 `imgs/` 子目录，并用相对路径引用。
- 引用论文、博客或外部资源时，尽量给出来源链接或论文信息。
- 论文阅读类文章建议包含：解决的问题、方法、关键实验/现象、个人理解、总结。
- 对推测性内容要明确标注为个人推测、经验判断或待验证结论。
- 使用 Docusaurus admonition 时遵循现有风格，例如 `:::tip`、`:::warning`。
- 数学公式可用 `$...$` 或 `$$...$$`，不要引入新的数学渲染方案。

## Markdown / MDX 注意事项

- `.md` 用于普通文档，`.mdx` 用于需要 JSX/React 组件的文档。
- MDX 中 JSX 标签需要正确闭合。
- Markdown 表格、代码块、列表之间保持空行，避免 Docusaurus 解析异常。
- 代码块必须标注语言，例如：

````markdown
```python
print("hello")
```
````

- 包含特殊字符的标题如果需要被稳定链接，建议显式添加锚点或保持标题稳定。
- 不要在文档中提交本地绝对路径、私有 token、账号凭据或未授权资料。

## React / TypeScript 代码约定

自定义组件主要位于 `src/components/`、`src/pages/` 和 `src/theme/`。

现有代码风格：

- 使用 TypeScript + React 函数组件。
- Docusaurus 相关导入使用 `@docusaurus/*`、`@theme/*`、`@site/*` 别名。
- 页面和组件可使用 CSS Modules，例如 `index.module.css`、`styles.module.css`。
- 全局样式放在 `src/css/custom.css`。
- 组件返回类型可显式写为 `JSX.Element`。
- 样式拼接可使用 `clsx`。

开发约定：

- 优先复用已有 Docusaurus 组件，如 `Layout`、`Link`、`Heading`。
- 新增交互组件前先检查是否已有相似组件可复用。
- 不要无必要引入新的 UI 框架或运行时依赖；如确需新增依赖，应说明原因。
- 修改 `src/theme/` 下 swizzle 组件时要谨慎，避免破坏全站文档和博客渲染。
- 评论相关逻辑在 `src/components/GiscusComponent.tsx`，涉及 repoId、categoryId 等配置时不要随意替换。

## 侧边栏与导航约定

- 默认 docs 的侧边栏在 `sidebars.ts`。
- LLMs 独立文档区侧边栏在 `sidebarsLlms.ts`。
- Paper Reading 独立文档区侧边栏在 `sidebarsPaperReading.ts`。
- Weekly Paper 独立文档区侧边栏在 `sidebarsWeeklyPaper.ts`。
- 当前侧边栏大量使用 `autogenerated`，因此新增文档时通常只需放入正确目录，并设置好 `sidebar_position` 或文件名前缀。
- 如果新增一级栏目，需要同步修改 sidebar 和 `docusaurus.config.ts` 中的 navbar。
- 不要随意改动已有 slug，否则会导致历史链接失效。

## 静态资源与图片约定

- 文档内图片优先放在同级或当前主题目录的 `imgs/` 文件夹。
- 站点级公共资源放在 `static/`，引用时从站点根路径开始。
- 图片文件名建议使用英文小写、数字、连字符。
- 新增图片后应运行构建检查路径是否正确。
- 不要提交过大的二进制文件；论文 PDF 已有部分放在文档图片目录中，新增前需确认是否必要。

## 部署流程

GitHub Actions 配置位于 `.github/workflows/deploy.yaml`：

- push 到 `master` 分支触发。
- 使用 Node.js 20。
- 安装依赖后运行站点构建。
- 构建产物 `build/` 部署到 GitHub Pages。

修改构建、依赖、Docusaurus 配置或文档路由后，应至少运行：

```bash
npm run typecheck
npm run build
```

## AI 助手工作准则

在本仓库中工作时，请遵守以下流程：

1. 先识别改动类型：文档内容、站点配置、React 组件、样式、部署配置或依赖。
2. 修改前先阅读相关目录的现有文件，保持命名、结构、front matter 和写作风格一致。
3. 文档改动优先保持中文表达自然、结构清晰，不要过度改写用户已有观点。
4. 代码改动优先使用现有依赖和 Docusaurus 官方模式，不要凭空引入新框架。
5. 对链接、slug、sidebar、navbar 的改动要特别谨慎，避免破坏已有页面路径。
6. 涉及配置、组件或依赖改动后，运行 `npm run typecheck` 和 `npm run build`。
7. 仅文档内容小改也建议运行 `npm run build`，用于发现 Markdown/MDX 解析和资源路径问题。
8. 不要提交 secrets、私有 token、个人账号凭据或未确认可公开的资料。
9. 不要自动提交 git commit，除非用户明确要求。

## 常见任务指南

### 新增一篇 docs 文档

1. 确认应放入 `docs/` 还是 `llms/`。
2. 选择对应主题目录，必要时创建子目录和 `_category_.yml`。
3. 使用 `.md`；如果需要 React/JSX 再使用 `.mdx`。
4. 添加合适的 front matter，例如：

```markdown
---
sidebar_position: 1
slug: topic-slug
---
```

5. 图片放入同级 `imgs/`，使用相对路径引用。
6. 运行 `npm run build` 检查页面生成。

### 新增一篇论文阅读博客

1. 放入 `blog/YYYY/MM/`。
2. 使用包含 slug、title、authors、tags 的 front matter。
3. 建议结构：
   - 论文信息
   - 解决的问题
   - 方法
   - 关键细节
   - 实验或结果
   - 个人理解
   - 总结
4. 对个人推测和论文原文结论做区分。

### 修改首页或组件

1. 首页位于 `src/pages/index.tsx`。
2. 首页模块位于 `src/components/HomepageFeatures/`。
3. 样式使用同目录 CSS Module 或全局 `src/css/custom.css`。
4. 修改后运行 `npm run typecheck` 和 `npm run build`。

### 修改导航或文档结构

1. 默认 docs 修改 `sidebars.ts`。
2. llms 文档区修改 `sidebarsLlms.ts`。
3. 顶部导航修改 `docusaurus.config.ts` 的 `themeConfig.navbar.items`。
4. 避免改动已发布页面 slug；确需改动时应考虑重定向方案。
